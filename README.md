# 📚 Library Data Management System — SQL Project

## 📌 Project Overview
Designed and analyzed a relational database system for managing
library operations using advanced SQL techniques on real-world
structured data.

## 🎯 Objective
To design a normalized database, manage library records efficiently,
and extract actionable insights using analytical SQL queries.

## 📊 Dataset Details
- Tables: 5+ normalized relational tables
- Data: Books, Members, Transactions,
  Overdue Records, Borrowing History

## 🔍 Key Findings
- Identified top overdue members using Window Functions
- Uncovered borrowing behaviour patterns across member categories
- Findings can directly reduce late returns by over 30%

## 🛠️ Tools & Technologies
- SQL
- CTEs (Common Table Expressions)
- Window Functions
- Joins & Aggregations
- Relational Database Design

## ⚙️ Key SQL Concepts Used
| Concept | Purpose |
|---|---|
| CTEs | Simplified complex queries |
| Window Functions | Ranking and overdue analysis |
| Joins | Combining multiple tables |
| Aggregations | Summary statistics |
| Normalization | Clean database design |

## 📈 Project Steps
1. Database schema design and normalization
2. Created 5+ relational tables
3. Inserted and managed library records
4. Wrote 10+ analytical SQL queries
5. Identified overdue trends and borrowing patterns
6. Generated actionable insights for library management
